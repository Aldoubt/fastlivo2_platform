# FAST-Calib 标定采集流程

本文档记录在本机使用 `tools/FAST-Calib-ROS2` 对 LiDAR-Camera 外参进行标定的完整流程。FAST-Calib 是基于标定板的雷达-相机外参标定方法，输入是一段静态点云 rosbag2 和同一时刻的清晰图像，输出可直接写入 FAST-LIVO2 配置的 `Rcl`、`Pcl`。

当前本机主要验证数据采集、参数配置和 ROS 2 运行链路。

## 1. 数据要求

一次标定至少准备以下两类数据：

- 一段 5 到 6 秒的 LiDAR 点云 rosbag2，bag 中需要包含 `sensor_msgs/msg/PointCloud2` 类型点云话题。
- 一张与该 bag 同一静态场景对应的相机图像，图像中标定板边缘、圆孔和 ArUco/Tag 标签必须清晰可见。

推荐目录结构：

```bash
/home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/calib_data/<case_name>/
  metadata.yaml
  <case_name>_0.db3
  image.png
```

其中 `<case_name>` 可按日期或场景命名，例如 `mid360_20260705_001`。

## 2. 采集前检查

启动 LiDAR 和相机驱动后，先确认话题存在：

```bash
source /opt/ros/humble/setup.bash
source /home/jetson/fastlivo2_platform/ros2_ws/install/setup.bash
ros2 topic list -t --no-daemon
```

重点确认：

- `/livox/lidar` 存在，且类型为 `sensor_msgs/msg/PointCloud2`。
- `/left_camera/image` 存在，画面中能看到完整标定板。

注意：FAST-Calib ROS2 当前读取的是 rosbag2 中的 `PointCloud2`，不要使用 Livox `CustomMsg` 格式作为标定 bag 输入。采集标定数据时如果启动 Livox 驱动，优先使用能发布 `PointCloud2` 的配置。

## 3. 标定板摆放

标定板需要同时被 LiDAR 和相机完整观测到。

- 标定板保持静止，采集过程中不要移动传感器和标定板。
- 标定板尽量正对相机，但可以有少量倾角，确保圆孔边缘在点云中有清晰轮廓。
- 标定板距离建议先放在 `1.5 m` 到 `3.0 m` 范围内，对应当前 `qr_params.yaml` 默认距离滤波。
- 图像中 ArUco/Tag 不能过曝、模糊或被遮挡。
- 点云中标定板前后不要有太近的杂物，否则会影响平面分割和圆心提取。

如果一次结果不稳定，建议在不同距离和角度采集多组静态数据，每组仍保持 5 到 6 秒。

## 4. 录制 LiDAR 标定 bag

在仓库根目录执行：

```bash
cd /home/jetson/fastlivo2_platform
mkdir -p tools/FAST-Calib-ROS2/calib_data

ros2 bag record \
  -o tools/FAST-Calib-ROS2/calib_data/<case_name> \
  /livox/lidar
```

录制 5 到 6 秒后按 `Ctrl+C` 停止。录制完成后检查 bag 信息：

```bash
ros2 bag info tools/FAST-Calib-ROS2/calib_data/<case_name>
```

确认 bag 中有 `/livox/lidar`，消息数量合理，并且类型是 `sensor_msgs/msg/PointCloud2`。

## 5. 保存同步图像

在录制 bag 的同一静态场景中保存一张清晰图像到同一个数据目录，例如：

```bash
tools/FAST-Calib-ROS2/calib_data/<case_name>/image.png
```

保存方式可以使用相机工具、`rqt_image_view` 截图，或从同步录制的相机 bag 中抽帧。关键要求是：图像必须对应同一次静态摆放，不要求与 LiDAR 点云严格逐帧同步，但场景不能变化。

图像质量检查：

- 标定板完整出现在画面中。
- ArUco/Tag 四角清晰可检测。
- 圆孔、板边缘没有明显运动模糊。
- 图像分辨率与相机内参对应。

## 6. 相机内参标定

FAST-Calib 需要先知道相机内参。ROS 2 里有现成工具 `camera_calibration`，常用命令是 `ros2 run camera_calibration cameracalibrator`，适合普通棋盘格内参标定。

### 6.1 准备棋盘格

准备一张平整棋盘格标定板，并量准单个格子的边长，单位用米。命令里的 `--size` 填的是内角点数量，不是棋盘格方块数量。

例如棋盘格是 `9 x 6` 个内角点，单格边长是 `0.025 m`，则后续使用：

```bash
--size 9x6 --square 0.025
```

标定采集建议：

- 棋盘格覆盖画面中心、四角、边缘等不同位置。
- 改变棋盘格距离和姿态，包含正对、倾斜、旋转等角度。
- 避免反光、过曝、运动模糊。
- 标定时相机分辨率、镜头焦距、曝光配置要与后续 FAST-Calib 使用时一致。

### 6.2 启动相机

先启动相机驱动，并确认图像话题存在：

```bash
source /opt/ros/humble/setup.bash
source /home/jetson/fastlivo2_platform/ros2_ws/install/setup.bash
ros2 launch hik_camera_ros2_driver hik_camera_launch.py
```

另开终端检查：

```bash
source /home/jetson/fastlivo2_platform/ros2_ws/install/setup.bash
ros2 topic list -t --no-daemon
```

本项目默认图像话题是：

```bash
/left_camera/image
```

相机驱动当前从以下文件加载内参：

```bash
/home/jetson/fastlivo2_platform/ros2_ws/src/hik_camera_ros2_driver/config/camera_info.yaml
```

对应参数配置在：

```bash
/home/jetson/fastlivo2_platform/ros2_ws/src/hik_camera_ros2_driver/config/camera_params.yaml
```

### 6.3 运行内参标定工具

确认系统已安装 `camera_calibration` 后运行：

```bash
source /opt/ros/humble/setup.bash
source /home/jetson/fastlivo2_platform/ros2_ws/install/setup.bash

ros2 run camera_calibration cameracalibrator \
  --size 9x6 \
  --square 0.025 \
  --ros-args \
  -r image:=/left_camera/image \
  -r camera:=/left_camera
```

如果当前版本的工具不接受 `--ros-args` 写法，可以使用兼容写法：

```bash
ros2 run camera_calibration cameracalibrator \
  --size 9x6 \
  --square 0.025 \
  image:=/left_camera/image \
  camera:=/left_camera
```

如果提示找不到包，安装 ROS 2 Humble 的相机标定工具：

```bash
sudo apt install ros-humble-camera-calibration
```

运行后移动棋盘格，让工具采集到足够多的角点样本。界面中 `X`、`Y`、`Size`、`Skew` 进度尽量覆盖充分后，点击 `CALIBRATE`，等待优化完成；结果满意后点击 `SAVE` 或 `COMMIT`。

说明：

- `SAVE` 通常会在 `/tmp` 下保存一个标定结果压缩包或 YAML 文件。
- `COMMIT` 会尝试通过 ROS 相机信息服务写回驱动；如果当前驱动不支持写回服务，就使用 `SAVE` 的结果手动更新配置。

### 6.4 手动整理内参结果

将标定结果中的 `camera_matrix` 和 `distortion_coefficients` 对应到 FAST-Calib 参数：

```yaml
camera_matrix:
  data: [fx, 0, cx,
         0, fy, cy,
         0, 0, 1]
distortion_coefficients:
  data: [k1, k2, p1, p2, k3]
```

写入 `tools/FAST-Calib-ROS2/config/qr_params.yaml` 时只需要前四个畸变参数：

```yaml
fx: <fx>
fy: <fy>
cx: <cx>
cy: <cy>
k1: <k1>
k2: <k2>
p1: <p1>
p2: <p2>
```

同时建议把完整结果更新到相机驱动的 `camera_info.yaml`，至少包含：

```yaml
image_width: <width>
image_height: <height>
camera_name: left_camera
camera_matrix:
  rows: 3
  cols: 3
  data: [fx, 0.0, cx, 0.0, fy, cy, 0.0, 0.0, 1.0]
distortion_model: plumb_bob
distortion_coefficients:
  rows: 1
  cols: 5
  data: [k1, k2, p1, p2, k3]
```

`projection_matrix` 可以使用标定工具输出的 `P` 矩阵。若只用于未矫正原图和 FAST-Calib，关键是 `camera_matrix`、`distortion_coefficients` 与图像分辨率正确一致。

### 6.5 内参验收

内参标定完成后，建议检查：

- 标定工具输出的 reprojection error 越小越好，普通镜头通常应尽量低于 `0.5 px` 到 `1.0 px`。
- 棋盘格角点重投影应贴合实际角点。
- 用新 `camera_info.yaml` 重启相机后，`/left_camera/camera_info` 中的 `K`、`D` 与标定结果一致。
- FAST-Calib 使用的图像分辨率必须与内参标定时一致。

## 7. 修改 FAST-Calib 参数

参数文件：

```bash
/home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/config/qr_params.yaml
```

需要重点修改四类参数。

### 7.1 相机内参

将当前相机内参和畸变参数写入：

```yaml
fx: <camera_fx>
fy: <camera_fy>
cx: <camera_cx>
cy: <camera_cy>
k1: <camera_k1>
k2: <camera_k2>
p1: <camera_p1>
p2: <camera_p2>
```

这些参数必须对应保存图像的分辨率。如果相机输出分辨率变了，需要使用该分辨率下的内参。

### 7.2 标定板尺寸

根据实际打印和制作尺寸修改：

```yaml
marker_size: 0.20
delta_width_qr_center: 0.55
delta_height_qr_center: 0.35
delta_width_circles: 0.5
delta_height_circles: 0.4
circle_radius: 0.12
```

含义：

- `marker_size`：单个 ArUco/Tag 的边长，单位米。
- `delta_width_qr_center`：左右两个 marker 中心的水平半间距，单位米。
- `delta_height_qr_center`：上下两个 marker 中心的竖直半间距，单位米。
- `delta_width_circles`：两个圆心的水平距离，单位米。
- `delta_height_circles`：两个圆心的竖直距离，单位米。
- `circle_radius`：圆孔半径，单位米。

这里的尺寸必须以实际标定板为准，打印缩放会直接影响外参精度。

### 7.3 点云距离滤波

根据标定板在 LiDAR 坐标系下的大致范围设置：

```yaml
x_min: 1.5
x_max: 3.0
y_min: -1.5
y_max: 2.0
z_min: -0.5
z_max: 2.0
```

目标是保留标定板点云，同时尽量去掉地面、墙面和其他杂物。多保留一点无关点通常可以接受，但如果出现大面积干扰平面，应收紧范围。

### 7.4 输入输出路径

将路径改为本次数据：

```yaml
lidar_topic: "/livox/lidar"
bag_path: "/home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/calib_data/<case_name>"
image_path: "/home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/calib_data/<case_name>/image.png"
output_path: "/home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/output"
```

`bag_path` 填 rosbag2 目录，不要填 `.db3` 文件路径。

## 8. 运行 FAST-Calib

本机已经提供启动脚本：

```bash
/home/jetson/fastlivo2_platform/tools/run_fast_calib.sh
```

等价手动命令：

```bash
source /opt/ros/humble/setup.bash
source /home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/install/setup.bash
export LD_PRELOAD=/usr/lib/aarch64-linux-gnu/libusb-1.0.so.0
ros2 launch fast_calib calib.launch.py
```

默认会启动 RViz。若只想运行算法，可使用：

```bash
ros2 launch fast_calib calib.launch.py rviz:=false
```

运行结束后查看输出目录：

```bash
ls -lh /home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/output
```

主要结果文件：

- `calib_result.txt`：FAST-LIVO2 可用的相机内参、`Rcl`、`Pcl`。
- `qr_detect.png`：图像中 marker 检测结果。
- `filtered_cloud.ply`：距离滤波后的点云。
- `plane_cloud.ply`：提取到的标定板平面。
- `edge_cloud.ply`：圆孔边缘点。
- `colored_cloud.pcd` / `colored_cloud.ply`：按图像上色后的点云，用于检查外参效果。

## 9. 结果验收

先打开 `calib_result.txt`：

```bash
sed -n '1,80p' /home/jetson/fastlivo2_platform/tools/FAST-Calib-ROS2/output/calib_result.txt
```

结果格式示例：

```yaml
Rcl: [ -0.029990,  -0.999537,   0.005183,
        0.021624,  -0.005833,  -0.999749,
        0.999316,  -0.029870,   0.021789]
Pcl: [ -0.003662,  -0.096965,  -0.045516]
```

验收重点：

- `qr_detect.png` 中 marker 检测位置正确。
- `plane_cloud.ply` 主要是标定板平面，不应被墙面或地面替代。
- `edge_cloud.ply` 中圆孔边缘提取完整。
- `colored_cloud.ply` 上色后，点云边缘应与图像中的标定板边缘、圆孔位置对齐。
- 多组数据标定出的 `Rcl`、`Pcl` 应接近；如果差异很大，优先检查标定板尺寸、内参、滤波范围和图像清晰度。

## 10. 写入 FAST-LIVO2 配置

将 `calib_result.txt` 中的 `Rcl` 和 `Pcl` 写入 FAST-LIVO2 对应配置文件的 `extrin_calib` 段，例如：

```bash
/home/jetson/fastlivo2_platform/ros2_ws/src/FAST-LIVO2/config/avia.yaml
```

对应位置：

```yaml
extrin_calib:
  extrinsic_T: [...]
  extrinsic_R: [...]
  Rcl: [...]
  Pcl: [...]
```

只替换 `Rcl` 和 `Pcl`。`extrinsic_T`、`extrinsic_R` 是 LiDAR-IMU 外参，不是 FAST-Calib 输出的相机-LiDAR 外参，除非另有 IMU 标定结果，不要混填。

写入后重新启动 FAST-LIVO2 验证建图和上色效果：

```bash
source /home/jetson/fastlivo2_platform/ros2_ws/install/setup.bash
ros2 launch fast_livo mapping_aviz.launch.py use_rviz:=True
```

## 11. 常见问题

### 11.1 读不到 bag

- `bag_path` 必须是 rosbag2 目录。
- 目录下应包含 `metadata.yaml` 和 `.db3` 文件。
- 点云话题名需要与 `lidar_topic` 一致。

### 11.2 检测不到 marker

- 检查 `marker_size` 是否与实际尺寸一致。
- 检查图像是否过曝、模糊、遮挡或分辨率不匹配。
- 确认使用的是 FAST-Calib 支持的标定板样式。

### 11.3 点云圆孔提取失败

- 收紧或调整 `x_min/x_max/y_min/y_max/z_min/z_max`，让滤波后主要保留标定板。
- 增加标定板与背景的距离，避免墙面或其他平面干扰。
- 确认标定板距离在 LiDAR 有效测距范围内，圆孔边缘点足够密。

### 11.4 上色点云明显错位

- 首先检查相机内参是否对应当前图像分辨率。
- 再检查标定板尺寸是否按米填写且没有打印缩放误差。
- 对比多组静态数据结果，如果只有单组异常，重新采集该组数据。
- 确认写入 FAST-LIVO2 时没有把 `Rcl/Pcl` 与 `extrinsic_R/extrinsic_T` 混淆。

## 12. 最小执行清单

1. 启动 LiDAR 和相机，确认 `/livox/lidar` 为 `PointCloud2`。
2. 如果相机内参未知或分辨率变更，先用 `camera_calibration` 完成内参标定。
3. 将内参写入 `tools/FAST-Calib-ROS2/config/qr_params.yaml`，并同步更新相机驱动 `camera_info.yaml`。
4. 固定 FAST-Calib 标定板，录制 5 到 6 秒 `/livox/lidar` rosbag2。
5. 保存同一静态场景下的清晰图像到数据目录。
6. 修改 `qr_params.yaml` 的标定板尺寸、滤波范围、输入输出路径。
7. 运行 `tools/run_fast_calib.sh`。
8. 检查 `output/calib_result.txt`、`qr_detect.png`、`colored_cloud.ply`。
9. 将 `Rcl`、`Pcl` 写入 FAST-LIVO2 配置并重启验证。
